# クラウド同期のセットアップ手順（管理者向け）

このアプリのデータを**全員のスマホ・PCで共有**するための初回設定です。
一度だけ行えば、以後は各端末でパスコードを入れるだけで使えます。

- 所要時間：10〜15分
- 必要なもの：Google アカウント（無料）
- 費用：無料（Firebase の無料枠で十分。目安は下記「無料枠について」参照）

---

## ステップ1：Firebase プロジェクトを作る

1. ブラウザで https://console.firebase.google.com を開き、Google アカウントでログイン
2. 「**プロジェクトを作成**」をクリック
3. プロジェクト名に `kinko-sync` など好きな名前を入力して「続行」
4. 「このプロジェクトで Google アナリティクスを有効にする」は **オフ** にして「プロジェクトを作成」
5. 作成が終わったら「続行」

## ステップ2：Realtime Database を作る

1. 画面左のメニューから「**構築**」→「**Realtime Database**」を選ぶ
2. 「**データベースを作成**」をクリック
3. ロケーションは「**asia-southeast1（シンガポール）**」を選ぶ
   （日本リージョンは選べません。シンガポールが日本から一番近い選択肢です）
4. セキュリティルールは「**ロックモードで開始**」を選んで「有効にする」

## ステップ3：セキュリティルールを設定する

1. Realtime Database の画面上部の「**ルール**」タブを開く
2. 表示されている内容をすべて消し、下記をそのまま貼り付ける：

```json
{
  "rules": {
    "stores": {
      "$storeId": {
        ".read": true,
        ".write": true
      }
    }
  }
}
```

3. 「**公開**」ボタンを押す（警告が出ますが、そのまま公開してOKです）

> **この設定の意味**：データは `stores/（長い秘密のID）` という場所に保存されます。
> この秘密のIDはアプリに入力する「共有パスコード」から作られ、
> IDの一覧を外部から見ることはできません。つまり**パスコードを知らない人は
> データにたどり着けない**、という仕組みで保護されます。
> 銀行レベルの堅牢さではありませんが、店舗の現金メモの用途には十分です。
> パスコードは推測されにくいもの（8文字以上）にしてください。

## ステップ4：データベースの URL をコピーする

1. Realtime Database の「**データ**」タブを開く
2. 画面上部に表示されている URL をコピーする
   （例：`https://kinko-sync-default-rtdb.asia-southeast1.firebasedatabase.app`）

## ステップ5：アプリに URL を設定する

GitHub のリポジトリ（Cash-Handling）の `index.html` にある次の1行に、
ステップ4でコピーした URL を貼り付けます：

```js
const SYNC_DB_URL = '';
      ↓ こう書き換える
const SYNC_DB_URL = 'https://kinko-sync-default-rtdb.asia-southeast1.firebasedatabase.app';
```

**やり方（GitHub の画面上で編集する場合）**：

1. https://github.com/takemiyu2008-ops/Cash-Handling を開いてログイン
2. `index.html` をクリック → 右上の鉛筆アイコン（Edit）をクリック
3. Ctrl+F（Macは⌘+F）で `SYNC_DB_URL` を検索し、`''` の中に URL を貼り付ける
4. 右上の「**Commit changes...**」→ そのまま「Commit changes」で保存

> この作業は Claude に「SYNC_DB_URL にこの URL を設定して push して」と
> 頼んでもかまいません。

保存から数分後、公開ページ（https://takemiyu2008-ops.github.io/Cash-Handling/）に反映されます。

## ステップ6：各端末でパスコードを設定する

1. 各スマホ・PC でアプリのページを**開き直す**（開きっぱなしなら再読み込み）
2. 画面左上の「**同期未設定**」バッジ、または履歴タブの「**クラウド同期設定**」ボタンを押す
3. 店で決めた**共有パスコード**（全員同じもの・8文字以上推奨）を入力して「接続する」
4. バッジが「**⇅ 同期済み**」になれば完了。以後は自動で同期されます

- 最初に接続した端末のデータがクラウドに登録され、あとから接続した端末はそれを受け取ります
- 両方の端末に記録がある場合は、自動で統合されます（記録が消えることはありません）

---

## 無料枠について

Firebase の無料プラン（Spark）は「保存 1GB・通信 月10GB」まで無料です。
このアプリの記録は1件あたり約200バイトなので、**毎日数十件記録しても何年分も余裕**があります。
アプリ側も通信量を抑える作りにしてあるため、通常の使い方で無料枠を超えることはまずありません。

## 困ったときは

| 症状 | 対処 |
|---|---|
| バッジが「⚠ 同期エラー」のまま | URL の設定ミスの可能性。ステップ5の URL を確認 |
| バッジが「⇅ オフライン」 | 電波のない場所でも記録は端末に保存され、電波が戻れば自動で同期されます |
| 端末によって見えるデータが違う | その端末のパスコードが違う可能性。「クラウド同期設定」から一度解除し、正しいパスコードで接続し直す |
| パスコードを変えたい | 全端末で解除→新パスコードで接続し直し。ただし旧パスコードのデータは残るため、最初にデータのある端末から接続すること |
